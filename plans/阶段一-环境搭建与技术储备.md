### 1.1 学习 Docker 基础（3 天，自习日集中攻克）

**为什么放在最前面**：你不会 Docker，而后续所有部署都依赖它。不先学，后面每一步都会被卡住。

**学习内容**：

| 概念 | 要学到什么程度 |
|------|---------------|
| 镜像 (Image) | 理解镜像=模板；会 `docker pull python:3.11-slim` |
| 容器 (Container) | 理解容器=运行实例；会 `docker run -it python:3.11-slim bash` |
| Dockerfile | 会写：FROM → WORKDIR → COPY → RUN pip install → CMD |
| Volume 挂载 | 理解 `-v ./本地:/容器内`，知道为什么需要挂载（持久化、热更新） |
| 端口映射 | 理解 `-p 8000:8000`（主机端口:容器端口） |
| docker-compose.yml | 会写多服务编排：backend + frontend + chromadb |
| 常用命令 | `docker ps`、`docker logs`、`docker-compose up -d`、`docker-compose down` |

**实操步骤**（在云服务器上做）：
1. SSH 登录云服务器
2. `docker run hello-world` 验证 Docker 可用
3. 写一个最简单的 Dockerfile：基于 python:3.11，运行一个打印 "hello docker" 的脚本
4. 构建并运行：`docker build -t test . && docker run test`
5. 写 docker-compose.yml：两个服务（一个 Python web，一个 nginx）
6. `docker-compose up` 看两个服务同时启动

**交付物检查**：
- [ ] 能在云服务器上独立写 Dockerfile 和 docker-compose.yml
- [ ] 理解 Volume 的作用（数据不随容器删除而丢失）
- [ ] 理解端口映射的原理

**学习资源**：
- Docker 官方 Get Started（约 2 小时读完）
- 在云服务器上实操，不要只看不练

| 概念 | 要学到什么程度 |

| 依赖注入 | 了解 `Depends()` 的用法，不深入 |
| Swagger 文档 | 知道 `/docs` 自动生成交互式 API 文档 |

**需要写出的 3 个接口**：

```python
# 1. 健康检查

def health():
    return {"status": "ok", "api_key_configured": settings.api_key_configured}

# 2. 文件上传（只接收文件，暂不处理内容）
@router.post("/upload")

    return {"filename": file.filename, "size": len(await file.read())}

# 3. 查询接口（模拟，暂不接 RAG）
```

### 1.3 学习 Streamlit 基础（1.5 天，利用上课日晚间）

**学习内容**：

| 概念 | 要学到什么程度 |
|------|---------------|
| 文本组件 | `st.title`、`st.markdown`、`st.text` |
| 输入组件 | `st.text_input`、`st.file_uploader` |
| 聊天组件 | `st.chat_message`、`st.chat_input` |
| 状态管理 | `st.session_state`（页面刷新不丢失数据） |
| 侧边栏 | `st.sidebar` |

**需要写出的最小聊天界面**：

```python
import streamlit as st

st.title("RAG 知识库问答")

# 初始化消息历史
if "messages" not in st.session_state:
    st.session_state.messages = []

# 显示历史消息
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.write(msg["content"])

# 用户输入
if question := st.chat_input("请输入你的问题"):
    st.session_state.messages.append({"role": "user", "content": question})
    with st.chat_message("user"):
        st.write(question)

    # 模拟 AI 回复
    answer = f"这是对「{question}」的模拟回答（实际回答将在阶段二接入 RAG）"
    with st.chat_message("assistant"):
        st.write(answer)
    st.session_state.messages.append({"role": "assistant", "content": answer})
```

**交付物检查**：
- [ ] `streamlit run app.py` 可启动，浏览器可交互
- [ ] 输入问题后显示模拟回答
- [ ] 刷新页面后对话历史仍然存在
- [ ] 代码提交到 Git

**学习资源**：
- Streamlit 官方 Tutorial 前 4 章

### 1.4 Git 分支与提交规范（0.5 天）

**学习内容**：

| 操作 | 命令 |
|------|------|
| 创建分支 | `git checkout -b feat/xxx` |
| 切换分支 | `git checkout main` |
| 合并分支 | `git merge feat/xxx` |
| 查看历史 | `git log --oneline --graph` |

**本项目 Git 规范**：

- **分支命名**：`feat/功能名`（新功能）、`fix/问题名`（修 bug）、`docs/内容`（文档）
- **Commit 格式**：`[阶段X] 简短描述——具体做了什么`
  - 例：`[阶段二] 实现文档解析器——支持 PDF/Word/TXT，含扫描件 OCR`
  - 例：`[阶段三] FastAPI 问答接口——支持流式输出 SSE`
- **Tags**：每个阶段结束打 tag（v0.1、v0.2...）

**交付物检查**：

**API 连通性测试脚本**（存为 `backend/test_api_connection.py`）：

```python
"""DeepSeek API 连通性测试 —— 运行此脚本验证 API Key 是否可用"""
import os
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()

client = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url=os.getenv("DEEPSEEK_BASE_URL", "https://api.deepseek.com"),
)

# 测试 Chat API
response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[{"role": "user", "content": "你好，请用一句话介绍你自己"}],
    max_tokens=100,
)
print("✅ Chat API 连通成功！")
print(f"回复: {response.choices[0].message.content}")
print(f"Token 消耗: {response.usage.total_tokens}")

# 测试 Embedding API（仅模型名不同，调用方式与 OpenAI 一致）
emb_response = client.embeddings.create(
    model="deepseek-embedding",  # 或 text-embedding-3-small 等兼容模型
    input=["测试文本"],
)
print(f"\n✅ Embedding API 连通成功！")
print(f"向量维度: {len(emb_response.data[0].embedding)}")
```

## 阶段一常见阻塞点

| 阻塞 | 解决方式 |
|------|----------|
| Docker 概念不理解 | 不要只看文档，直接动手：docker run、docker build，跑起来再理解 |
| FastAPI 报错看不懂 | 先看 Swagger（/docs），它能直接测试接口并显示错误信息 |
| Streamlit 页面刷新后数据丢失 | 用 `st.session_state` 存储需要持久化的数据 |
| API 测试脚本报错 401 | API Key 没填或填错，检查 `.env` 文件 |
| 云服务器连不上 | 检查安全组/防火墙是否开放了 SSH 端口（通常是 22） |
