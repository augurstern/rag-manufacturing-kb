## 每个阶段的核心交付

### 阶段一
- Docker 能独立写 Dockerfile 和 docker-compose.yml
- FastAPI 三个基础接口 + Swagger
- Streamlit 聊天界面（模拟回答）
- Git 仓库初始化

### 阶段二 ⭐ 最重要
- `parser.py` — 文档解析（PDF/Word/TXT/扫描件）
- `chunker.py` — 三种分块策略（滑动窗口/父子/表格行）
- `embedder.py` — DeepSeek Embedding + ChromaDB
- `retriever.py` — 向量+BM25+RRF 混合检索
- `generator.py` — Prompt 模板 + 幻觉抑制 + 来源引用
- CLI 验证：5-8 份文档，20 个问题，准确率 ≥80%

### 阶段三
- FastAPI 4 组路由（upload/query/history/documents）
- SQLite 审计日志（documents 表 + audit_logs 表）
- Streamlit 4 页面（首页/上传/聊天/历史）
- API 集成测试 + 联调测试报告

### 阶段四
- Dockerfile x2 + docker-compose.yml
- 云服务器部署，公网 IP 可访问
- 配置安全（文件白名单/限流/日志）
- 性能测试报告 + README 完善

### 阶段五
- 架构设计文档 + 模块实现笔记
- 面试话术（30秒/2分钟/5分钟）
- 18 道高频面试题回答要点
- 2 轮模拟面试 + 复盘
- Demo 演示视频
- 简历项目描述定稿
- 仓库最终检查 → v1.0

## 使用方法

1. **每个阶段开始前**：通读该阶段的计划文档，了解任务清单和交付物
2. **每个阶段进行中**：对照任务清单逐项完成，在 check box 里打勾
3. **每个阶段结束时**：在文档末尾填写实际完成日期、用时、遇到的困难
4. **面试前**：回顾阶段二的"设计决策记录"和阶段五的"面试话术"

## 关键提示

- **阶段二最重要**：面试 80% 的技术问题出在这里。写代码的同时记设计决策笔记。
- **不要追求完美**：先跑通再优化。能讲出"为什么这样设计"比"代码写得多好"更重要。
- **Git 提交要分散**：commit 历史要体现 80 天的开发过程，不能集中在几天提交完。
- **技术选型可调整**：如果遇到阻塞（如 DeepSeek API 不稳定），可以切换备选方案，同步更新计划。
