# 四项目综合时间线

**总周期**：2026.06.30 — 2026.09.20（83 天）
**策略**：串行为主、后期小幅重叠。前 3 周只做 RAG，7/20 后 RAG 为主 + 补课为辅，8/15 后逐步启动后续项目。

---

## 为什么不是四个同时做

1. **时间碎片化**：上课日晚间只有 3 小时连块，同时维护 4 个项目 = 每项目每周 ~7h，形不成连续思考
2. **学习依赖链**：Docker/FastAPI → RAG → API 调用模式 → Agent → Agent 概念 → 多Agent → 模型理解 → LoRA，跳过前面直接做后面会不断被基础卡住
3. **共用 API 调试混乱**：四个项目都调 DeepSeek API，同时开发时很难定位问题
4. **Git 历史暴露**：时间戳集中在 7-8 月，不像正常工作节奏

## 关键原则

1. **永远先保 RAG**：如果某个阶段时间不够，压缩的是项目二/三/四，不是 RAG
2. **补课可以提前，代码不能提前**：7/20 就开始看 ReAct 文档，但代码 8/15 才写。因为提前看了有四周的背景消化时间，写起来更快
3. **仓库提交节奏要模拟真实**：每个项目的 commit 分散在数周内，不要集中一个周末全部提交
4. **如果进度慢于计划**：砍项目四（只需 3 天）最后做，再不行砍项目三（只留设计文档），保前两个
